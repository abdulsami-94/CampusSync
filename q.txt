## PYTHON BASICS

**Q: What is Python?**
A: Python is a high-level, interpreted programming language known for its simplicity and readability. It supports multiple programming paradigms including procedural, object-oriented, and functional programming.

**Q: What are Python data types?**
A: Basic data types: int, float, str, bool, list, tuple, dict, set.

**Q: What is a function in Python?**
A: A function is a block of reusable code that performs a specific task. Defined using 'def' keyword.

**Q: What is a class in Python?**
A: A class is a blueprint for creating objects. It defines attributes and methods.

**Q: What is inheritance in Python?**
A: Inheritance allows a class to inherit attributes and methods from another class.

**Q: What are decorators in Python?**
A: Decorators are functions that modify the behavior of other functions or classes.

**Q: What is a virtual environment?**
A: A virtual environment is an isolated Python environment that allows you to install packages for a specific project without affecting the global Python installation.

## FLASK FRAMEWORK

**Q: What is Flask?**
A: Flask is a lightweight WSGI web application framework in Python. It's designed to make getting started quick and easy, with the ability to scale up to complex applications.

**Q: What is WSGI?**
A: Web Server Gateway Interface - a specification for how web servers communicate with web applications.

**Q: How do you create a Flask application?**
A: Import Flask, create an app instance: app = Flask(__name__)

**Q: What is routing in Flask?**
A: Routing maps URLs to Python functions using the @app.route() decorator.

**Q: What are Flask blueprints?**
A: Blueprints organize Flask applications into modules, allowing for better code organization.

**Q: What is Jinja2?**
A: Jinja2 is Flask's default template engine for rendering HTML templates.

**Q: What is Flask-WTF?**
A: Flask-WTF integrates WTForms with Flask for form handling and validation.

**Q: What is Flask-Login?**
A: Flask-Login provides user session management for Flask applications.

**Q: What is SQLAlchemy?**
A: SQLAlchemy is a Python SQL toolkit and Object-Relational Mapping (ORM) library.

## CAMPUSSYNC PROJECT

**Q: What is CampusSync?**
A: CampusSync is a Flask-based web application for managing student complaints in ASM CSIT College. It provides role-based access for students, staff, and administrators.

**Q: What are the main features of CampusSync?**
A: Role-based access control, complaint submission with file uploads, admin dashboard with analytics, staff assignment system, upvoting and commenting on complaints, responsive UI.

**Q: What are the user roles in CampusSync?**
A: Three roles: Student, Staff, and Admin.

**Q: What can students do in CampusSync?**
A: Register, submit complaints with images, upvote and comment on complaints, view and edit their own complaints.

**Q: What can staff do?**
A: View assigned complaints, update complaint status from In Progress to Resolved.

**Q: What can admins do?**
A: View all complaints, assign complaints to staff, delete complaints (soft delete), view analytics dashboard.

**Q: What is the complaint workflow?**
A: Pending → In Progress (assigned to staff) → Resolved.

**Q: What database is used?**
A: SQLite for development and production.

**Q: What ORM is used?**
A: SQLAlchemy with Flask-SQLAlchemy extension.

**Q: What are the main models in CampusSync?**
A: User, Complaint, Upvote, Comment.

**Q: What is soft delete?**
A: Soft delete marks records as deleted without actually removing them from the database, preserving data integrity.

**Q: How are passwords stored?**
A: Passwords are hashed using bcrypt before storage.

**Q: What is CSRF protection?**
A: Cross-Site Request Forgery protection prevents unauthorized form submissions.

**Q: How are file uploads handled?**
A: Files are saved with UUID names for security, only image types allowed, maximum 16MB.

**Q: What frontend technologies are used?**
A: Vanilla CSS, HTML5, Chart.js for analytics, Bootstrap for responsive design.

**Q: How is authentication implemented?**
A: Using Flask-Login for session management and Flask-Bcrypt for password hashing.

**Q: What is the application factory pattern?**
A: A pattern where the Flask app is created in a function, allowing for better testing and configuration management.

**Q: How is the app deployed?**
A: Using Gunicorn as WSGI server for production deployment.

**Q: What are the complaint categories?**
A: Wi-Fi/Internet, Classrooms, Computer Labs, Canteen, Washrooms, Library, Hostel, Sports Facilities, Electricity, Other.

**Q: What are complaint priorities?**
A: Low, Medium, High.

**Q: What are complaint statuses?**
A: Pending, In Progress, Resolved.

**Q: How are analytics displayed?**
A: Using Chart.js for pie charts (status breakdown) and bar charts (category breakdown).

**Q: What is pagination?**
A: Pagination divides large datasets into smaller pages for better performance and user experience.

**Q: Why restrict emails to @asmedu.org?**
A: To ensure only ASM CSIT College students and staff can register, maintaining institutional security.

## TECHNICAL QUESTIONS

**Q: Explain the Flask app structure.**
A: Application factory in __init__.py, blueprints for different modules (auth, admin, staff, student), models in models.py, config in config.py.

**Q: How does user authentication work?**
A: User logs in, Flask-Login creates session, user_loader function loads user from database by ID.

**Q: Explain the database relationships.**
A: User has many complaints (submitted), User has many assigned_complaints (as staff), Complaint belongs to User (author) and optionally to User (assignee).

**Q: How are forms validated?**
A: Using Flask-WTF with WTForms validators, custom validators for email domain restriction.

**Q: How does soft delete work?**
A: is_deleted boolean field, queries filter out deleted records unless explicitly requested.

**Q: Explain file upload process.**
A: Form validates file type and size, generates UUID filename, saves to uploads folder, stores filename in database.

**Q: How are analytics generated?**
A: SQLAlchemy queries count complaints by status and category, data passed to Chart.js for visualization.

**Q: What is the deployment process?**
A: Install dependencies, set environment variables, run with Gunicorn, configure for production.

**Q: How do you handle errors?**
A: Custom error pages (403, 404, 500), try-except blocks, proper error logging.

**Q: Explain the seed_db.py script.**
A: Creates demo users and complaints for testing, uses app context to interact with database.
